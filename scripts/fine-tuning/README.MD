# Дообучение модели-эмбеддера

Изолированный пайплайн для дообучения моделей-эмбеддеров. Используется метод генерации синтетических QA-пар и контрастивное обучение с MultipleNegativesRankingLoss.

Модель доступна по ссылке: https://huggingface.co/shezoff/voproshalych-bge-m3

## Структура пайплайна

Пайплайн разбит на 3 независимых этапа:

1. **`synthetic_dataset.py`** – Асинхронный генератор датасета. Обращается к LLM (через OpenRouter) и генерирует вопросы студентов на основе чанков сырого текста.
2. **`deduplicate.py`** – Модуль семантической очистки. Отсеивает вопросы со косинусным сходством > 85%, чтобы избежать переобучения модели на дубликатах.
3. **`train.py`** – Скрипт полного дообучения базовой модели `deepvk/USER-bge-m3`. 

## Требования к железу
* **GPU:** Обучение настроено для работы на локальной видеокарте. Параметр `BATCH_SIZE = 8` в `train.py` оптимизирован под 8 ГБ VRAM.
* **RAM:** Рекомендуется от 16 ГБ оперативной памяти для загрузки модели и векторизации датасета.

## Быстрый старт

Находясь в папке `scripts/fine-tuning/`

### 1. Установка окружения

```bash
python -m venv .venv
.venv\Scripts\activate  # Для Windows
pip install -r requirements.txt
```

### 2. Настройка ключей

Скопируйте шаблон переменных окружения:

```bash
cp .env.example .env
```

В файле `.env` обязательно укажите `OPENROUTER_API_KEY` для работы генератора датасета.

### 3. Подготовка данных

Поместите исходный файл с текстами базы знаний в корень текущей папки под именем `chunks_result.txt` (разделитель чанков: `<<<DELIMITER>>>`).

### 4. Запуск пайплайна

Запускайте скрипты строго по очереди:

```bash
# Шаг 1: Генерация (на выходе qa_dataset.jsonl)
python synthetic_dataset.py

# Шаг 2: Очистка (на выходе qa_dataset_semantic_clean.jsonl)
python deduplicate.py

# Шаг 3: Обучение (на выходе папка finetuned_bge_m3_v1/)
python train.py

```

Готовые веса дообученной модели сохраняются локально. В архитектуре RAG-ассистента достаточно указать путь к директории `finetuned_bge_m3_v1/` при инициализации `SentenceTransformer`.
